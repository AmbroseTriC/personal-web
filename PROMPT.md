---

## Step 1: Design Direction

You are building a personal website for an Economics student at UCL with finance internship experience. The site must feel like it was designed by a real human with taste — not generated by AI.

### Choose ONE bold aesthetic and commit fully. Suggested directions (pick or invent your own):

- **Editorial / Magazine** — asymmetric layouts, strong typographic hierarchy, newspaper-grid energy, dramatic whitespace
- **Swiss / International Typographic** — strict grid, bold sans-serif headlines, restrained color, information-dense
- **Neo-Brutalist** — raw borders, monospace accents, high contrast, visible structure, anti-polish
- **Analog / Tactile** — paper textures, stamp-like elements, hand-drawn borders, warm tones, print-inspired

Whatever you choose, the result must be visually distinctive and impossible to confuse with a generic AI template.

### Typography

- Pick a **characterful display font** for headings (from Google Fonts — e.g., DM Serif Display, Playfair Display, Bricolage Grotesque, Instrument Serif, Fraunces, or something unexpected)
- Pair it with a **clean but non-generic body font** (e.g., DM Sans, Source Serif 4, Literata, IBM Plex Sans)
- Do NOT use Inter, Roboto, system-ui, Poppins, Space Grotesk, Outfit, or Sora

### Color

- Build a **custom palette** of 3–5 colors derived from the chosen aesthetic — not from Tailwind defaults
- Define colors as CSS custom properties
- Ensure sufficient contrast for accessibility (WCAG AA minimum)

## Step 2: Content Sections

Build these sections using ONLY the data from `Profile.pdf`:

1. **Header / Hero** — Name and a one-line identity (e.g., "Economics at UCL"). No fake taglines, no "passionate about..." filler. Keep it factual and confident.
2. **About** — A brief, honest summary. You may lightly infer tone (e.g., London-based economics student with finance experience) but do NOT invent hobbies, skills, or interests not in the PDF.
3. **Experience** — List roles from the PDF with company, title, dates, and location. No fabricated bullet points or responsibilities.
4. **Education** — Degrees and institutions from the PDF.
5. **Contact** — Email and LinkedIn from the PDF. Provide as clickable links.

---

## Step 3: Technical Requirements

- **Single file**: `index.html` — all CSS inlined in `<style>`, no external stylesheets (Google Fonts via `<link>` is fine)
- **No JavaScript frameworks**. Vanilla JS only, and only if needed for a specific interaction (e.g., a subtle scroll effect). No JS is also acceptable.
- **Responsive**: must look good on mobile (375px), tablet (768px), and desktop (1440px)
- **Semantic HTML**: use `<header>`, `<main>`, `<section>`, `<footer>`, `<article>`, `<nav>` appropriately
- **Performance**: no heavy assets, no external images, no icon libraries. If you need visual elements, use CSS-only shapes, borders, or unicode characters.
- **Accessibility**: proper heading hierarchy, sufficient color contrast, focusable links, alt text where relevant

## Step 4: Anti-Pattern Enforcement

Re-read `anti-patterns-blacklist.md` and verify the final output against EVERY rule:

- [ ] No purple/blue/pink gradients
- [ ] No Inter/Roboto/system-ui/Poppins/Space Grotesk fonts
- [ ] No raw Tailwind default colors
- [ ] No 3-column identical card grid
- [ ] No centered hero with "Get Started" + "Learn More" buttons

- [ ] No gratuitous glassmorphism or backdrop-blur

- [ ] No floating gradient blob decorations
